# 操作手冊：把 spike App 裝到 iPhone 並實測背景定位

要回答的問題只有一個：**iOS 在鎖屏、App 切到背景之後，會不會把持續定位停掉。**

程式已經寫好、CI 已經產出可安裝的 IPA。以下是你要做的部分。
**AI 做不了的就是這一段**——它需要你的實體 iPhone 與 USB 線。

---

## IPA 已經幫你抓好了

```
D:\Project\WayinkIosSpike\dist\WayinkLocationSpike-ipa\WayinkLocationSpike.ipa
```

（若要自己重抓：GitHub repo → Actions → 點最新一次成功的執行 → 下方 Artifacts。
注意**下載回來是一個 zip**，裡面才是 `.ipa`，要先解壓縮。）

---

## 一、Windows 端準備（約 20 分鐘，只需做一次）

1. **安裝 Apple 官網版 iTunes**
   ⚠️ **不可以用 Microsoft Store 版**。Store 版是沙箱化的，Sideloadly 抓不到它需要的底層元件。
   若你已經裝了 Store 版，要先移除再裝官網版。
   下載處：<https://www.apple.com/itunes/download/win64>

2. **安裝 Sideloadly**：<https://sideloadly.io>（Windows 版，免費）

---

## 二、側載到 iPhone（約 10 分鐘）

3. iPhone 用 USB 接上電腦，手機上出現「信任這台電腦？」時選**信任**。

4. 開 Sideloadly：
   - `IPA` 欄位選上面那個 `.ipa` 檔
   - `Apple account` 填你的 **Apple ID**（免費的就可以，不需要付 99 美元）
   - 按 **Start**，輸入密碼
   - ⚠️ 若你的 Apple ID 有開兩步驗證，**這裡要填 app-specific password**，不是你平常的密碼。
     到 <https://account.apple.com> → 登入 → 「App 專用密碼」產生一組。

5. **iPhone：信任開發者憑證**
   設定 → 一般 → VPN 與裝置管理 → 點你的 Apple ID → **信任**
   （沒做這步的話 App 點下去會直接閃退）

6. **iPhone：開啟開發者模式**
   設定 → 隱私權與安全性 → 最下面的 **開發者模式** → 開啟 → **依提示重新開機**
   ⚠️ **iOS 16 一定要做這步。漏掉會以為是側載失敗**，其實只是沒開。

---

## 三、開始測之前，先看一眼首頁最上面那張卡

App 一打開，最上面就是「背景模式」卡片，它直接讀取**實際安裝在你手機上那份** Info.plist：

- ✅ **綠色「location 已保留」** → 可以往下測
- ❌ **紅色「缺少 location」** → **停，不要出門**。代表 Sideloadly 重簽時把背景模式拿掉了，
  這種狀態下走一小時也是白走。這是整條路唯一沒有官方保證的環節，所以特地做成第一眼就能看到。

---

## 四、權限要給到「永遠」

7. 按「開始記錄」，iOS 會跳權限詢問。
   **iOS 不會第一次就給你「永遠」的選項**，它只會給「使用 App 期間」。這是正常的。

8. 給完之後到：設定 → 隱私權與安全性 → 定位服務 → **Wayink Spike** → 選 **永遠**
   順便把同一頁的 **精確位置** 打開。

9. 回到 App，確認「定位權限」顯示 **永遠**、狀態顯示 **追蹤中**。

---

## 五、先做 5 分鐘短測（不要跳過）

**理由**：出門一小時是有成本的，先用 5 分鐘確認整條鏈路真的在動，不要把錯誤留到一小時後才發現。

10. 確認畫面上的筆數正在增加。
11. **按電源鍵鎖屏**，把手機放口袋，在附近走 3~5 分鐘。
12. 回來解鎖看 App：

- **筆數明顯增加了** → 背景定位在動，可以做完整測試 ✅
- **筆數停在鎖屏當下** → 背景定位沒生效，先別出門，把畫面上顯示的內容跟我說

---

## 六、一小時完整測試

13. 按開始、鎖屏、放口袋，**正常出門走一小時**（走路即可，不必特地繞路）。
    中途**不要**特地點開 App 看——那會把它拉回前景，就測不到「背景能撐多久」了。

14. 回來後打開 App，看兩個數字：
    - **筆數**：走一小時、每秒一筆的話會是數千筆等級
    - **最後一筆**：如果顯示「3 秒前」，代表**到剛剛都還在記** ✅
      如果顯示「43 分鐘前」，代表**它在 43 分鐘前就被停掉了** ❌ ——這就是我們要抓的東西

---

## 七、把記錄檔給我

兩種方式，挑順手的：

- **A（最快）**：App 裡的「分享記錄檔」按鈕 → 用 LINE／郵件傳給自己，再貼給我
- **B**：iPhone 的「檔案」App → 我的 iPhone → **Wayink Spike** → `wayink-spike-log.txt`

記錄檔長這樣：

```
2026-07-21T09:12:03+08:00 [前景] UPDATES_STARTED
2026-07-21T09:12:04+08:00 [前景] FIX lat=25.033000 lon=121.565000 acc=5.0 spd=1.2 age=0.1
2026-07-21T09:12:31+08:00 [背景] APP_BACKGROUND
2026-07-21T09:12:32+08:00 [背景] FIX lat=25.033100 lon=121.565100 acc=8.0 spd=1.4 age=0.2
```

中括號裡是當下 App 的狀態。**看到 `[背景]` 開頭的 `FIX` 行，就代表背景定位真的在運作。**
若中間出現 `SYSTEM_PAUSED`，那是 iOS 自己喊停，是本次實驗最想抓到的證據。

## 已知限制

- **免費 Apple ID 簽的 App 7 天後會失效**，開起來會閃退。要再測就用 Sideloadly 重裝一次即可
  （測一小時完全不受影響）。
- 免費帳號同時最多側載 3 個 App。
- **repo 目前是 private**，GitHub Actions 的 macOS runner 會吃 **10 倍**配額，
  Free 方案 2000 分鐘實際只剩約 200 分鐘、大約 5~6 次建置。
  之後若要頻繁改，到 repo Settings 把它改成 **public** 就完全免費、無上限
  （內容只有這支測試 App，不含 Wayink 任何程式碼或金鑰）。
